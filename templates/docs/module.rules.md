# Module Template Rules

该规则文件约束 `modules/<task_id>.md` 的写法。

## 适用场景

- 单个教学单元
- 单个模块
- 单条职责边界清晰的分析任务

## 强制要求

1. 必须有引言，直接说明本节在解决什么学习问题。
2. 目录不可省略。
3. 至少包含 2 个代码块。
4. 每个代码块都必须包含：
   - 为什么看这段代码
   - 代码本身
   - 对代码的具体解释
5. `critical_flow` 必须是顺着真实代码走的一条链路，而不是抽象流程图描述。
6. 文末必须有返回总索引链接。

## 解释深度要求

- 解释应足够详细，让第一次接触项目的读者能跟下来。
- 解释必须客观，不得把推测当成事实。
- 每个结论最好能点明对应的源码位置或文件路径。

## 常见失败模式

- 把“核心对象与职责”写成文件列表
- 有代码块但没有解释
- 解释只有结论，没有说明为什么能从代码里得出结论
- “建议继续阅读”没有具体文档入口
